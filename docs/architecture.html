03 Data Flow — Dashboard Summary

One concrete request traced end to end, from mount to render.

Dashboard.vue mounts and calls useFilters().getCurrentFilters(), reading the 4 shared filter refs (period, warehouse, category, status).
api.getDashboardSummary(filters) in api.js builds a URLSearchParams query, dropping any filter still set to 'all'.
Axios sends GET http://localhost:8001/api/dashboard/summary?warehouse=...&category=....
FastAPI's get_dashboard_summary() reads the in-memory inventory_items and orders lists, applies apply_filters() then filter_by_month().
Aggregates are computed — total_inventory_value, low_stock_items, pending_orders, total_backlog_items, total_orders_value — and returned as JSON.
Axios resolves response.data; Dashboard.vue renders stat cards and charts directly from it — no client-side caching layer.

04 Filter System

Four filters — Time Period, Warehouse, Category, Order Status — apply uniformly wherever the backend supports them. State lives once, shared everywhere.

Frontend — useFilters.js

  • Four module-level singleton refs, defaulting to 'all' — shared across every view, not per-component state.
  • FilterBar.vue binds directly to these refs via v-model.
  • getCurrentFilters() maps UI names to API params: selectedLocation → warehouse, selectedPeriod → month.

Backend — main.py

  • apply_filters() — exact match on warehouse, case-insensitive on category/status.
  • filter_by_month() — exact YYYY-MM match, or quarter codes (Q1-2025…) via a QUARTER_MAP.
  • /api/demand and /api/backlog ignore all filters — always return the full list.

06 Known Gaps

Found while tracing the frontend against the live route table — worth knowing before building on top of either.

Discrepancies
  • api.js and App.vue call /api/tasks (GET/POST/PATCH/DELETE) and /api/purchase-orders — none of these routes exist in main.py. Calls 404 at runtime; the frontend's try/catch silently swallows the failure.
  • purchase_orders.json is loaded but empty ([]) — only read inside /api/backlog to compute has_purchase_order.
  • Backlog.vue exists in views/ but isn't registered in main.js's router — unreachable from the main nav.
  • Reports.vue bypasses the centralized api.js client and calls axios directly — inconsistent with every other view.
